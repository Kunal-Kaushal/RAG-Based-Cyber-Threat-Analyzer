Brute Force Detection Pattern: Multiple failed SSH login attempts from the same IP. Threshold: more than 5 failed attempts within 60 seconds from a single IP indicates brute force. Log pattern: "Failed password for [user] from [IP] port [port] ssh2". Common targeted usernames: root, admin, ubuntu, pi.
Brute Force Mitigation: Block offending IP using ufw deny from [IP] to any. Enable fail2ban with SSH jail. Use iptables -A INPUT -s [IP] -j DROP. Enforce multi-factor authentication. Limit SSH access to known IPs only.
Brute Force Escalation: If same IP shows failed passwords followed by successful login, immediate account compromise is likely. Severity escalates from High to Critical. Rotate all credentials immediately and audit session activity.
DDoS Detection Pattern: Single or multiple IPs sending more than 1000 requests per second. High frequency GET requests from same IP or IP range. SYN flood shows large number of half-open connections. UDP flood shows massive UDP packets overwhelming bandwidth.
DDoS Mitigation: Rate limit with nginx limit_req_zone. Null route attacking IP range at router level. Enable Cloudflare DDoS protection. Use iptables rate limiting rules. Contact upstream ISP for traffic scrubbing during volumetric attacks.
Port Scan Detection Pattern: Single IP attempting connections to more than 10 different ports within 30 seconds. SYN scan shows connection attempts with no completion. Sequential port hits indicate automated scanning tool like nmap. UDP scans show multiple ICMP port unreachable responses.
Port Scan Mitigation: Block scanning IP immediately. Close all unused ports. Deploy honeypot ports to detect and trap scanners early. Use iptables to drop packets from offending IP. Enable port knocking for sensitive services.
Kill Chain Pattern: Port scan from IP X followed within 10 minutes by brute force from same IP X indicates targeted attack. Severity is Critical. Immediate block recommended. Attacker is performing reconnaissance then attempting access.
Credential Stuffing Pattern: Differs from brute force — uses different username and password combinations from breach databases. Log shows many different usernames failing from same IP. Mitigation includes IP blocking, CAPTCHA, and breach password detection.
Privilege Escalation Pattern: User account suddenly accessing admin resources or running sudo commands not previously used. Changes to /etc/passwd or /etc/sudoers indicate active privilege escalation. Severity is Critical. Isolate affected account immediately.
Data Exfiltration Pattern: Large outbound data transfers at unusual hours. Multiple connections to unknown external IPs. Repeated DNS queries to uncommon domains. High volume outbound traffic on unusual destination ports like 4444 or 6666.
Reverse Shell Indicator: Outbound connection initiated from server to external IP on non-standard port. Common ports used: 4444, 1234, 9001. Unexpected outbound connection from internal server requires immediate isolation of affected system.
SSH Tunneling Pattern: SSH connections with unusually long session durations. High data transfer over SSH from non-admin users. Indicates data exfiltration or persistent access channel being maintained by attacker.
Repeat Offender Pattern: Same IP address appearing in logs across multiple attack types or multiple days indicates persistent threat actor. Severity should be escalated to Critical regardless of individual attack type. Permanent IP block and upstream reporting recommended.
Log4Shell Exploitation Pattern: Requests containing jndi ldap strings in HTTP headers or parameters. Affects systems running Log4j versions before 2.15.0. Severity is Critical. Patch immediately and scan all Java applications.
